FINNA-APP API ENDPOINT SPECIFICATION
====================================
Source: /root/projects/finna-app/backend/app/api
Last Updated: 2026-04-26

API BASE URL: /api/v1
AUTH: JWT Bearer token required on all /api/v1/* endpoints except:
- GET  /healthz (no auth)
- POST /api/v1/auth/token (auth endpoint)  

================================================================================
SECTION 1: API ROUTES WITH HTTP METHOD, PATH, PARAMETERS
================================================================================

--- AUTHENTICATION ---
1. POST /api/v1/auth/token
   Description: Obtain JWT access token
   Request Body:
     - username: string (required)
     - password: string (required)
   Response:
     - JSON: { "token": "jwt_token_string" }
   Status Codes: 200, 401, 403

2. POST /api/v1/auth/gcp/register
   Description: Register GCP project credentials
   Auth: Required
   Request Body:
     - project_id: string (required) - GCP project ID
     - key_file_content: string (optional) - Service account key JSON
   Response:
     - JSON: { "config_id": "uuid", "project_id": "string" }
   Status Codes: 200, 401

3. POST /api/v1/auth/azure/service-account
   Description: Register Azure service principal credentials
   Auth: Required
   Request Body:
     - tenant_id: string (required)
     - client_id: string (required)
     - client_secret: string (optional)
     - subscription_id: string (optional)
   Response:
     - JSON: { "config_id": "uuid", "message": "success message" }
   Status Codes: 200, 401

4. POST /api/v1/auth/login
   Description: CLI-compatible alias for /api/v1/auth/token
   Request Body:
     - username: string (required)
     - password: string (required)
   Response: { "access_token": "jwt_token", "token_type": "bearer" }
   Status Codes: 200, 401, 403


--- CONFIGURATION MANAGEMENT ---
5. GET /api/v1/config
   Description: List all cloud configurations
   Auth: Required
   Response: Array of CloudConfigResponse objects
   Status Codes: 200, 401, 422

6. POST /api/v1/config
   Description: Create a new cloud configuration
   Auth: Required
   Request Body (CloudConfigCreate):
     - provider: "azure" | "gcp" (required)
     - name: string (required)
     - credential_type: CredentialType enum (required)
     - config: object (required)
   Response: CloudConfigResponse
   Status Codes: 201, 401

7. GET /api/v1/config/{config_id}
   Description: Get a specific cloud configuration
   Auth: Required
   Path Parameters:
     - config_id: string (UUID, required)
   Response: CloudConfigResponse
   Status Codes: 200, 404, 401

8. PUT /api/v1/config/{config_id}
   Description: Update a cloud configuration
   Auth: Required
   Path Parameters:
     - config_id: string (UUID, required)
   Request Body (CloudConfigUpdate):
     - name: string (optional)
     - credential_type: CredentialType enum (optional)
     - config: object (optional)
   Response: CloudConfigResponse
   Status Codes: 200, 400, 404, 401

9. DELETE /api/v1/config/{config_id}
   Description: Delete a cloud configuration
   Auth: Required
   Path Parameters:
     - config_id: string (UUID, required)
   Response: 204 No Content
   Status Codes: 204, 404, 401

10. GET /api/v1/config/provider/{provider}
    Description: List configurations filtered by provider
    Auth: Required
    Path Parameters:
      - provider: "azure" | "gcp" (required)
    Response: Array of CloudConfigResponse objects
    Status Codes: 200, 401

11. POST /api/v1/config/{config_id}/test
    Description: Test cloud credentials without extraction
    Auth: Required
    Path Parameters:
      - config_id: string (UUID, required)
    Request Body: None
    Response: ConfigTestResult
      - ok: boolean
      - provider: "azure" | "gcp"
      - token_expires_at: integer (Azure only)
      - checks: object with auth details
      - error: string (if failed)
    Status Codes: 200, 404, 401

12. GET /api/v1/config/projects
    Description: List all FinOps projects
    Auth: Required
    Response: Array of ProjectResponse objects
    Status Codes: 200, 401

13. POST /api/v1/config/projects
    Description: Create a new FinOps project
    Auth: Required
    Request Body:
      - name: string (required)
      - slug: string (optional)
      - owner: string (optional)
      - cost_center: string (optional)
      - budget_cap: number (optional)
      - tags: object (optional)
      - note: string (optional)
    Response: ProjectResponse with id
    Status Codes: 201, 401

SECTION 4: AUTHENTICATION REQUIREMENTS
================================================================================

Protected Endpoints (/api/v1/* requires Bearer token):
  1. All config endpoints (GET POST PUT DELETE /config/*)
  2. All extractors endpoints
  3. All costs endpoints
  4. All alerts endpoints
  5. All db (internal) endpoints

Public Endpoints (no auth required):
  1. GET  /healthz
  2. POST /api/v1/auth/token
  3. POST /api/v1/auth/login

Auth Flow:
  1. POST /api/v1/auth/token with {username, password}
  2. Receive token in response
  3. Include in Authorization header: "Bearer <token>"
  4. Token expires based on JWT_EXPIRATION_MINUTES env variable (default 60)


================================================================================
SECTION 5: RATE LIMITING
================================================================================

Configuration:
  - RATE_LIMIT_REQUESTS: 100 (requests per window)
  - RATE_LIMIT_WINDOW: 60 (seconds)

Headers returned on all requests:
  - X-RateLimit-Limit: 100
  - X-RateLimit-Remaining: <remaining count>
  - X-RateLimit-Reset: <unix timestamp>

Error Response (429):
  - { "detail": "Rate limit exceeded", "retry_after": 60 }
  - Retry-After: 60
  - X-RateLimit-Limit: 100
  - X-RateLimit-Remaining: 0
  - X-RateLimit-Reset: <timestamp>
